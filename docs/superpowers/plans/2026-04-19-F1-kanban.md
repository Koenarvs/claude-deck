## Recommended task order
1. Add dnd-kit deps.
2. Build `api.ts` with `apiGet<T>`, `apiPost<T>`, `apiPatch<T>`, `apiDelete<T>` — each validates response via a provided zod schema (or returns raw on schemas not defined).
3. TDD `KanbanCard`: props `{goal}`, renders title, tags, priority dot, model badge; click → `/goals/:id`.
4. TDD `KanbanColumn`: props `{status, goals}`, renders header + card list; drop zone for dnd.
5. TDD `KanbanBoard`: reads goalsByStatus for each of 4 statuses (planning, active, waiting, complete), renders 4 columns. Drag a card to different column → PATCH /api/goals/:id with new status and kanban_order.
6. TDD `NewGoalModal`: form with title, cwd (uses existing FolderBrowser from carry-over — wait for F6 carry-over OR inline a simple text input for v1), model picker, initial prompt textarea, permission mode toggle, tags input. Submit → POST /api/goals, optimistic upsert into store, close modal.
7. Wire into `KanbanPage`: load goals on mount (GET /api/goals) into store, render `<KanbanBoard>`, show `<NewGoalModal>` toggled by a button.

## Kanban order math
- On drop between two cards A (order=2.5) and B (order=3.5): new card gets (2.5+3.5)/2 = 3.0
- At column start: first_existing / 2 (min 0.5)
- At column end: last_existing + 1.0
- Moving between columns: same math, new status

## Acceptance criteria (spec §13.1 + §14.3 F1)
- [ ] 50 goals render < 200ms cold (use React DevTools profiler in dev)
- [ ] Drag a card to a different column → PATCH /api/goals/:id fires with new status + kanban_order; on 200, store updates; on error, rollback
- [ ] New Goal modal submit creates a goal via POST; UI updates optimistically; error rolls back
- [ ] Live: receiving `goal:created` or `goal:updated` WS event updates the board without refresh
- [ ] Clicking a card navigates to `/goals/:id`
- [ ] Empty columns show a placeholder "No goals in this status"
