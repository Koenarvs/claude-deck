All burst agents (B*, F*, S*) depend on F0 only. They do **not** depend on each other — interface contracts live in `src/shared/*` and are frozen by F0.

## Orchestration model

1. **Pre-flight smoke test (~30 min, solo):** Resolve the four open questions in spec §16 by running a real `claude` CLI session and inspecting actual stream-json events and hook payloads. Update the spec inline if findings change interfaces. Commit.
2. **Phase 1: F0 (~1-1.5 h, solo):** Dispatch one agent on the F0 plan. Review diff. Merge to main. **Gate: all Phase 2 blocks until F0 is on main.**
3. **Phase 2 (~3-4 h, parallel):** Dispatch 16 agents simultaneously. Each on its own branch `feat/<agent-id>`. Review as PRs come in.
4. **Phase 3: integration (~2-3 h, solo-supervised):** One session pulls all branches, merges burst-by-burst, runs tests, fixes drift, runs E2E.
5. **Smoke + tweak (~1-2 h):** Use the app end-to-end, spawn fix agents for rough edges.

## Agent teams (dev/QA pairing)

If using the experimental agent-teams feature, pair one QA agent per burst unit. See `agent-teams.md` for the pairing protocol. Each burst brief contains a **QA Checklist** section.

## Plan format conventions

- **F0** is written in full bite-sized TDD style — foundation quality matters most.
- **Burst briefs (B*, F*, S*)** are structured handoffs: scope, contracts, key files, recommended task order, AC-derived QA checklist, quality bar. Each dispatched dev agent uses `superpowers:test-driven-development` to decompose inside its brief.
- **Integration plan** is a serial runbook.

## Files the entire project depends on

These live in F0 and are immutable under burst agents:
- `src/shared/types.ts` — canonical type definitions
- `src/shared/schemas.ts` — zod validators
- `src/shared/events.ts` — WebSocket event unions
- `server/db/migrations/001_init.sql` — DB schema
- `package.json`, `tsconfig.json`, `vite.config.ts`, `tailwind.config`

If a burst agent discovers a gap in these files, they escalate rather than patch locally. The integrator resolves cross-cutting changes.
