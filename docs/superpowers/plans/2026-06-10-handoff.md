## Gotchas / lessons

- **Pre-existing flaky test:** `tests/client/pages/AnalyticsPage.test.tsx` "failed section shows an error message" — a `RouterFallback` race that fails **only** in the combined `npm test` run under worker contention; passes in isolation and in `--project client`/`--project server` alone. Not a logic regression. Worth fixing the component's `useInRouterContext` race separately.
- **Repo uses `exactOptionalPropertyTypes: true`** — don't set optional object props to `undefined`; use conditional spreads to omit them.
- **`tsx watch` auto-reloads** the live server on any `server/` edit — edits to the main tree restart it (kills live PTY sessions briefly). Migrations are idempotent (`CREATE TABLE IF NOT EXISTS`), safe on the live DB.
- Run **`npm run typecheck` before every commit** (one commit, `ec9cf07`, was committed red and fixed in `84996ec`).
