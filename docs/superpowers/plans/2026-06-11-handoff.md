### Security note (already resolved)
An automated review flagged the silent `allow` as a "supervised permission bypass." **It is not.**
`hooks/client.js` maps a non-deny response to **exit 0** (no `permissionDecision` emitted) → defers
to Claude Code's native permission flow. Supervised goals spawn in **default** mode
(`claude-adapter.ts:73` only adds `--permission-mode bypassPermissions` for **autonomous**), so risky
tools still prompt **natively in the terminal**. The dashboard was only ever an optional extra gate.
Documented inline at both hook returns.

## Branch topology & the "no-reload merge" technique (READ THIS)
The live server runs `tsx watch` in the **main checkout** on `feat/multi-agent-foundation`. Any
checkout-based merge rewrites those files → watcher reload → **live PTY sessions die**. To merge
without reloading, this session advanced the branch ref with **`git update-ref`** (moves the ref,
writes no files). That leaves the main checkout temporarily "ahead of disk" (HEAD ahead, working tree
old) — `git status` shows a scary inverse diff; it's harmless. Applying = `git reset --hard HEAD` in
the main checkout (materializes files + triggers the watcher reload). That apply has **already been
done** this session, so the tree is now consistent.

> If you add more commits intended for the live branch, repeat: commit on the worktree branch →
> `git update-ref refs/heads/feat/multi-agent-foundation <new> <old>` → (when the user okays a
> reload) `git reset --hard HEAD` in `C:/github/claude-deck`.
